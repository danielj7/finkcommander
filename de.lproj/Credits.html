Anerkennungen

Das FinkCommander Entwickler-Team möchte seinen Dank an folgende Personen aussprechen:

  • Zuallererst Christoph Pfisterer für die Erfindung von Fink, ohne das FinkCommander nichts als eine leere Tabelle und ein Haufen nutzloser Menüs und Schaltflächen wäre.

  • Max Horn und David Morrison dafür, dass sie nach dem Weggang von Christoph die Projektadministration so fähig weitergeführt haben.

  • Dave Love für den ursprünglichen und weiter verbesserten Code, der es FinkCommander endlich ermöglichte, das Security Framework statt sudo zu verwenden, um fink und apt-get Kommandos zu autorisieren.

  • Justin Hallett, Dave Vasilevsky und Brian Lenihan, die beim Debuggen der Probleme halfen, die FinkCommander mit Vorab-Versionen von Jaguar hatte. Ebenfalls hilfreich waren ihre Hinweise und Vorschläge zu zahlreichen anderen Teilen von FinkCommander.

  • Martin Costabel, der den Hinweis lieferte, wie die verfügbaren Pakete der binären Distribution von Fink ermittelt werden können.

  • Dem restlichen Fink Entwickler-Team dafür, dass sie Mac-Benutzern all diese grossartige Unix Software so einfach verfügbar gemacht haben.

Referenzen

Die folgenden Ressourcen waren äusserst hilfreich bei der Entwicklung von FinkCommander und werden sehr empfohlen:

  • Das Buch Cocoa Programming for Mac OS X von Aaron Hillegass. Es ist eine überragende Einführung in Cocoa.

  • Mike Beam's Cocoa tutorials, speziell die Tutorials über Werkzeugleisten und wie man Tabellenspalten anpasst, bei denen sich FinkCommander stark bedient.

  • Die Cocoa Dev Central Web Site, eine weitere sehr gute Quelle für Tutorials.

  • Die Open Source Code- und Programmier-Tips zu Cocoa Browser von Hoshi Takanori und Max Horn. FinkCommander verwendet die "MyTextView" Klasse von Cocoa Browser, um das Hand-Symbol über den Links im Paket-Inspektor erscheinen zu lassen.
