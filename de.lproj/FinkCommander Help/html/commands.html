Die Fink und Apt-Get Kommandos

Fink (Source Menü)

Die folgenden Erklärungen sind hauptsächlich der Fink Hilfeseite entnommen. Öffnen Sie ein Terminal und geben Sie "fink man" ein, um die komplette Version zu sehen. Falls Sie Fink nicht im Standardverzeichnis (/sw) installiert haben, sind alle Verweise auf /sw in diesem Text durch den Pfad Ihrer Fink Installation zu ersetzen.

Hauptmenü

Install

Dieses Kommando wird zur Installation von Paketen verwendet. Es lädt die Quelldateien herunter und konfiguriert, erstellt und installiert das ausgewählte Paket. Alle Pakete der sich ergebenden Abhängigkeiten werden ebenfalls installiert, nachdem Sie von Fink um Zustimmung gefragt wurden.

Reinstall

Dasselbe wie 'Install'. Jedoch installiert es das Paket via dpkg(8) auch wenn es bereits installiert sein sollte. Sie können diese Option verwenden, falls Sie irrtümlich Dateien des Paketes gelöscht haben oder wenn Sie die ursprüngliche Konfiguration des Paketes wiederherstellen wollen.

Build

Erstellt ein Paket ohne es zu installieren. Wie immer werden die Quelldateien heruntergeladen falls sie nicht bereits auf Ihrem System vorhanden sind. Als Resultat dieses Kommandos erhalten Sie eine zur Installation bereite .deb Datei. Falls diese Datei bereits vorhanden ist wird nichts ausgeführt. Es gilt zu bemerken, dass alle Pakete der sich ergebenden Abhängigkeiten nicht nur erstellt, sondern auch installiert werden.

Rebuild

Erstellt ein Paket (wie das Kommando 'Build'), aber ignoriert und überschreibt die möglicherweise bereits vorhandene .deb Datei des Paketes. Falls das Paket zurzeit installiert ist, bewirkt dieses Kommando zusätzlich eine Neuinstallation via dpkg(8). Hilfreich für die Entwicklung von Paketen.

Fetch

Lädt die Quelldatei des ausgewählten Paketes herunter, installiert dieses aber nicht. Dieses Kommando bewirkt immer das Herunterladen der Quelldatei, selbst wenn diese bereits auf Ihrem System vorhanden ist.

Remove

Entfernt Pakete durch den Aufruf des Kommandos dpkg --remove.

Das Kommando 'Remove' löscht die Dateien des Paketes, belässt aber die komprimierte .deb Datei auf Ihrem System. Somit können Sie zu einem späteren Zeitpunkt das Paket wieder installieren ohne es erneut kompilieren zu müssen. Falls sie Platz auf Ihrer Festplatte brauchen können Sie die .deb Datei aus dem Verzeichnis /sw/fink/dists, sowie die entsprechenden Verweise im Verzeichnis /sw/fink/debs löschen.

Selfupdate, Selfupdate-rsync, Selfupdate-cvs

Dieses Kommando automatisiert den Aktualisierungsprozess auf eine neue Version von Fink. Es überprüft die Website von Fink auf neue Versionen, lädt falls nötig die Quelldateien herunter und aktualisiert die Kernpakete, inklusive Fink selbst. Ohne die CVS Option wird nur auf offizielle neue Versionen von Fink aktualisiert, mit der CVS Option werden zusätzlich die Informationen über alle Pakete auf dem neusten Stand gehalten.

Bemerkung: Selfupdate-cvs verwendet automatisch CVS um die Informationen zu den neuesten Paketversionen zu erhalten.

Update-all

Dieses Kommando aktualisiert alle installierten Pakete auf die neueste Version.

Force remove

'Force remove' löscht ein Paket selbst wenn andere Pakete davon abhängig sind. Dieses Kommando sollte nur in letzter Instanz zur Lösung eines anderweitig nicht korrigierbaren Abhängigkeitsproblems verwendet werden.

Untermenü Andere

Cleanup

Entfernt veraltete .deb Dateien, Quellcode-Archive und verwaiste Verweise. Dieses Kommando befindet sich noch in Entwicklung und ist daher mit Vorsicht zu gebrauchen. Es können unter bestimmten Umständen Dateien entfernt werden, welche nicht veraltet sind oder auch veraltete Dateien auf Ihrem System belassen. Falls Ersteres passiert kann es sein, dass Sie gezwungen sind ein Paket erneut herunterzuladen oder zu kompilieren, welches Sie andernfalls vom .deb Archiv aus erneut hätten installieren können.

Index

Erstellt den Paket Zwischenspeicher neu. Gewöhnlicherweise sollte es nicht notwendig sein dieses Kommando manuell auszuführen, da Fink automatisch feststellt, wenn eine Aktualisierung nötig ist.

Describe

Dieses Kommando zeigt die Beschreibung des Paketes an, welches Sie auf der Kommandozeile eingeben. Anstatt das 'fink describe' Kommando aufzurufen, gibt es dieselbe Information durch Verwendung des von FinkCommander angelegten Zwischenspeichers aus.

Bemerkung: Das Kommando Werkzeuge:Paket-Inspektor und das Symbol 'Paket-Inspektor' in der Werkzeugliste zeigen dieselbe Information an wie das Kommando 'Describe' in einem separaten Fenster mit aktiven Hyperlinks zur Pakethomepage und E-Mail Adresse des Maintainers.

Apt-Get (Binär Menü)

Auschnitte des apt-get Manuals
Install

'Install' lädt das ausgewählte Paket selbst sowie alle Pakete der sich ergebenden Abhängigkeiten herunter und installiert sie auf Ihrem System.

Update descriptions

'Update descriptions' synchronisiert die Index Dateien der Pakete auf Ihrem System mit den Daten der erhältlichen Pakete. Vor 'update descriptions' sollte immer erst 'upgrade packages' durchgeführt werden.

Upgrade packages

'Upgrade packages' installiert die neuesten Versionen aller zurzeit auf Ihrem System installierten Pakete.
