Bekannte Fehler und Einschränkungen

Fehler bei Aufforderung zur Benutzereingabe

In einigen Fällen kann es vorkommen, dass die Aufforderung zur Benutzereingabe nicht angezeigt wird, wenn Fink auf eine solche Eingabe wartet. Falls dieses Problem auftritt, wählen Sie Werkzeuge:Eingabe an Fink senden, um eine Antwort einzugeben. Optional ist hierfür auch ein Symbol in der Werkzeugleiste vorhanden. Bitte senden Sie zusätzlich eine Email an sburrious@users.sourceforge.net mit der Beschreibung des Prompts, für den die Aufforderung zur Benutzereingabe nicht angezeigt wurde. Ich werde ihn in eine Liste von Prompts aufnehmen, die FinkCommander speziell beachtet, wenn Kommandos ausgeführt werden.

Fehler bei Aktualisierung mit apt-get

Falls Sie eine Fehlermeldung erhalten, nachdem Sie apt-get update ausgeführt haben, versuchen Sie erst Source:Scanpackages auszuführen und dann erneut apt-get update.

Atlas Installations Problem

Der Installationsvorgang des atlas Pakets beinhaltet einen langen interaktiven Teil, während dem mehrere Fragen auf der Kommandozeile beantwortet werden müssen. Wenn FinkCommander diesen Punkt erreicht bleibt das Programm stehen und gibt nicht einmal mehr die Eingabeaufforderungen aus. Ich bin nicht sicher wieso das passiert, aber es wäre ohnehin viel zu mühsam, all die Fragen mit Hilfe der FinkCommander Dialogaufforderung zur Benutzereingabe zu beantworten. Falls Sie also atlas installieren wollen, müssen Sie das entweder direkt im Terminal Programm oder über den FinkCommander Menübefehl Source:Im Terminal ausführen:Install tun.

Problem mit den Paketen tcsh and term-readkey-pm

Bei der Auswahl von tcsh oder term-readkey-pm wird eine Warnung ausgelöst, da FinkCommander bei der Installation dieser Pakete von Source hängen bleibt. Die Installation der Binär-Pakete und die Installation über die Kommandozeile funktioniert hingegen problemlos. Falls Sie dieses Problem auch mit anderen Paketen haben, erfassen Sie bitte einen Bug Report.
