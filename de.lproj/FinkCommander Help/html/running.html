Verwenden von Fink and Apt-Get Kommandos

Grundlagen

Wenn sich ein Kommando auf ein Paket bezieht (z.B. "Paket installieren"), dann markieren Sie ein oder mehrere Paket(e) in der Tabelle und wählen aus dem Source oder Binär Menü oder der Werkzeugleiste das Kommando, welches Sie ausführen möchten. Wie in jeder Tabellenauflistung können Sie Umstelltaste- oder Optionstaste-Klick verwenden, um mehr als ein Paket zu markieren. Sie können ebenfalls mehrere Pakete auswählen, indem Sie den Mauszeiger bei gedrückter linker Maustaste vertikal über die Paketliste ziehen (beim horizontalen Ziehen werden die .info and .patch Dateien eines Pakets kopiert oder geöffnet).

Um ein Kommando auszuführen, welches nicht paketabhängig ist (z.B. "Aktualisieren der Paketliste") wählen Sie einfach das Kommando aus dem Menü oder der Werkzeugleiste.

Die Werkzeugleiste stellt eine Anzahl Symbole zur Ausführung von Kommandos zur Verfügung, welche in der Grundkonfiguration nicht enthalten sind. Um die Werkzeugleiste Ihren Bedürfnissen anzupassen, öffnen Sie Darstellung:Symbolleiste anpassen. Generell sind Symbole für Operationen mit binären Paketen blau, solche für Operationen mit Paketen, welche direkt von Quelldateien erstellt werden, grün.

Wenn ein Befehl ausgeführt wird, erscheint auf der linken Seite der Statuszeile (die Zeile ganz unten im Hauptfenster) der Name des Kommandos und, falls dies der Platz erlaubt, eine Liste der Pakete auf welche der Befehl angewendet wird. Während Pakete von Quelldateien erstellt werden, erscheinen stattdessen Nachrichten, die die aktuelle Installationsphase bezeichnen, sowie rechts davon eine (deterministische) Fortschrittsanzeige. Sobald das Kommando beendet ist, kann es sein, dass in der Statuszeile die Meldung "Paketdaten werden jetzt aktualisiert" und rechts davon eine (nicht deterministischer) Fortschrittsanzeige erscheinen. Mehr Details.

Solange sich Kommandos in Ausführung befinden, werden die entsprechenden Menüeinträge und Werkzeugleisten-Symbole ausgeblendet. Es ist jedoch möglich, ein weiteres Kommando im Terminal auszuführen.

Kommandos zum Beenden von Prozessen

Sie können einen Prozess gewöhnlich beenden durch (1) Drücken des Stop-Symbols in der Werkzeugleiste, (2) Drücken von Option-. auf der Tastatur oder (3) Auswahl des Menüeintrags Werkzeuge:Befehl beenden.

Source vs. Binär

Die Kommandos im Menü Source werden durch den Fink Paket Manager ausgeführt, Kommandos im Menü Binär durch Apt-Get. Generell übernimmt Fink Pakete, welche als Quellcode heruntergeladen werden, Apt-Get Pakete, die in binärer Form verfügbar sind. Daher auch die Menünamen.

Zu jeder Regel gibt es natürlich auch Ausnahmen: Falls Sie zum Beispiel ein nur als Sourcecode erhältliches Paket installieren, es danach wieder entfernen und es erneut installieren, bevor eine neuere Version erhältlich wird, so verwendet der Fink Paket Manager das Kommando dpkg, da das Paket nun ja in binärer Form, als .deb Datei, auf Ihrem System vorliegt.

Befehle im Terminal ausführen

Wenn Sie einen Befehl aus dem Menü Source:Im Terminal ausführen oder dem Menü Binär:Im Terminal ausführen auswählen, so öffnet sich ein neues Terminalfenster, in welchem der ausgewählte Befehl ausgeführt wird. Möglicherweise werden Sie hierbei aufgefordert Ihr Passwort einzugeben. Dieselbe Option ist auch über das Kontextmenü der Pakettabelle und eine Anzahl zusätzlicher Werkzeugleisten-Symbole erreichbar.

Falls Sie wünschen können Sie diese Option verwenden, um ein zweites fink Kommando auszuführen, während ein anderes Kommando in FinkCommander ausgeführt wird. Setzen Sie diese Möglichkeit aber nur vorsichtig ein, da das gleichzeitige Ausführen von inkompatiblen Kommandos, wie z.B. das gleichzeitige Installieren eines Paketes und das Entfernen einer Abhängigkeit desselben Paketes, unvorhersehbare Folgen haben kann.
