Terminating Fink Processes

The FinkCommander terminate command searches for FinkCommander's descendant processes and runs sudo -S kill -KILL on each process. The -KILL signal forces a process to terminate without running any cleanup routines built into to the program's signal handlers. The -KILL signal was the only one I could find that would terminate Fink processes from FinkCommander with any reliability.

But occasionally, even this draconian approach doesn't work.

If the terminate command doesn't work, you will need to follow this procedure to stop the current fink process:
  1. Quit FinkCommander. The dialog following a failed terminate command will give you the option of doing this immediately.
  2. Open Terminal.
  3. Type in ps -acjx and look for processes toward the end of the resulting list that are owned by root and that appear to relate to the command you want to terminate, such as make, perl or cc. The process owner is identified in the first column of the list.
  4. Run sudo kill [process number] on the first of these, where process number is the the number in the second column of the list.
  5. Repeat steps 3 and 4 until all of the Fink related processes have terminated.
