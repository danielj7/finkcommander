Reading the Table

Installed, Binary, Unstable, Stable Columns

These columns show the appropriate version number for the category in question. By comparing the Binary version number with those in the other columns, you can tell whether running Binary:Install for a package will install the most recent available version or at least the most recent available stable version. See Running Commands:Source v. Binary for more details on the distinctions between source and binary commands.

The version numbers presented are those available for installation, taking into account whether you have enabled the installation of unstable packages.

Latest Column

This column serves primarily as a more concise alternative to the specific version columns described above. It shows the most recent version of the package available to you.

Status Column

"Current" means you have the most current available version of a package installed. "Outdated" means that the package is installed, but a more recent version is available. "Archived" means that you previously installed the package but removed it so that only the debian archive remains on your system or that you ran the build command for a package but have not yet installed it. (This information appeared in the Installed column in version 0.3.3a and earlier.)

Finding Packages

FinkCommander provides two handy tools for finding the packages you might be interested in installing or removing:

  1. You can sort the table by clicking on the column headers, just as you can in Mail and other applications. Click a header twice in a row to reverse the sort order.

  2. The toolbar in its default configuration includes a filter that can be applied to most of the fields in the table. The filter works incrementally, so that as you type or delete characters, the contents of the table change to include only those packages that satisfy your search criteria. The package count also changes to reflect the table contents.

By combining the use of sorting and filtering, you can effectively search the table with two criteria.

The table colums, by the way, can be resized and moved, and any changes you make will be saved between runtimes.
